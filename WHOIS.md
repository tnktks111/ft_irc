ft_irc: WHOISコマンド実装要件と設計まとめ
概要

WHOISコマンドは、特定のユーザーに関する詳細な情報を照会するために使用されるクエリコマンドです 。クライアント（特にirssiなど）は、プライベートメッセージの送信前やログイン直後の自己状態確認のためにこのコマンドを頻繁に使用します。ft_ircにおいて、リファレンスクライアントを正常に動作させ、公式のIRCサーバーと同等のユーザー体験を提供するために実装が強く推奨される機能です。
詳細解説
1. 基本構文とパラメータ (How)

コマンドは以下の構文で送信されます 。
Plaintext

WHOIS [ <target> ] <mask/nickname> *( "," <mask/nickname> )

    <target> パラメータ: クエリを特定のサーバーに転送するための引数です 。しかし、今回のft_ircではサーバー間通信の実装が禁止されているため 、この引数は無視するか、常に自サーバー宛てとして処理して問題ありません。

    <mask/nickname> パラメータ: 照会したい対象のニックネーム（複数指定時はカンマ区切り）を指定します 。

2. なぜWHOISが必要なのか (Why)

    クライアントのUI補完: ユーザー名、ホスト名、現在参加しているチャンネル一覧などの詳細なプロファイル情報を取得し、クライアント側の表示をリッチにするためです。

    自己診断（irssi特有）: irssiなどのクライアントは、ログイン完了直後に自分自身のニックネームに対してWHOISを自動送信し、サーバー側でどのようにホストマスクが解決されているかを確認します。

3. レスポンスの構築フロー (How)

対象ユーザーが見つかった場合、サーバーは複数のNumeric Replyを組み合わせて情報を送信します 。以下の順序で返却するのが一般的です。

    基本情報の送信: RPL_WHOISUSER (311) を使い、対象のユーザー名とホスト名を送ります 。

    接続サーバー情報の送信: RPL_WHOISSERVER (312) を使い、そのユーザーが接続しているサーバー名を送ります 。

    参加チャンネル一覧の送信: RPL_WHOISCHANNELS (319) を使い、ユーザーが現在参加しているチャンネルのリストを送ります（※未参加の場合は省略可） 。

    終了の宣言（必須）: 最後に必ず RPL_ENDOFWHOIS (318) を送信し、処理が完了したことをクライアントに知らせます 。

重要な用語 (Numeric Replies 一覧)

WHOISコマンドでハンドリングすべき主なステータスコードです 。
コード	マクロ名	説明とフォーマット例
311	RPL_WHOISUSER	

ユーザーの基本情報。

<nick> <user> <host> * :<real name>
312	RPL_WHOISSERVER	

接続先サーバー情報。

<nick> <server> :<server info>
319	RPL_WHOISCHANNELS	

参加中のチャンネル一覧。

<nick> :*( ( "@" / "+" ) <channel> " " )
318	RPL_ENDOFWHOIS	

リストの終了を通知。これを受け取るまでクライアントは待機します。

<nick> :End of WHOIS list
401	ERR_NOSUCHNICK	

指定されたニックネームがサーバー上に存在しない場合のエラー。

<nickname> :No such nick/channel
431	ERR_NONICKNAMEGIVEN	

ニックネームがパラメータとして与えられなかった場合のエラー。

:No nickname given
